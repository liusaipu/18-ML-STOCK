# 股票分析系统 — 机器学习预测模块设计文档

> 版本：v1.0  
> 日期：2026-04-07  
> 目标：基于「舆情+量价高频」与「财务趋势预警」双引擎，为个人投资者提供可解释的 AI 异动预警与基本面趋势判断。

---

## 一、总体架构

采用 **轻量双引擎** 架构：

- **引擎 A（主引擎）**：舆情 + 量价高频异动预警  
  预测未来 1~3 个交易日发生情绪驱动型异动的概率。
- **引擎 B（辅引擎）**：财务趋势恶化/改善预警  
  预测下一季度财报中核心财务指标的变化方向。

两者共同锚定目标：**预测「超预期事件」的概率**，而非精确股价。

---

## 二、引擎 A：舆情 + 量价高频异动预警

### 2.1 业务目标
不预测收盘价，而是预测 **未来 1~3 个交易日发生情绪驱动型异动** 的概率：
- 开盘跳空、放量突破、概念涨停潮跟风
- 每天收盘后给出明日早盘情绪评分
- 盘中每 30 分钟更新一次情绪-资金共振度

### 2.2 数据采集

| 数据源 | 频率 | 获取方式 | 价值 |
|--------|------|----------|------|
| 东方财富股吧热帖 | 5~15 分钟 | 爬取个股吧热门帖标题+摘要 | 散户情绪温度计 |
| 雪球/同花顺热门股榜 | 小时 | API 或页面解析 | 资金关注度排名 |
| 财经新闻快讯 | 实时 | 聚合 API（新浪/腾讯/财联社） | 突发政策/行业事件 |
| Level1 分时行情 | 1 分钟 | 腾讯/东方财富接口 | 量价背离、资金流向 |
| 概念板块异动 | 5 分钟 | 东方财富概念热度排行 | 板块效应跟风判断 |

### 2.3 特征工程

#### 文本侧：舆情向量化
使用轻量中文金融情感模型（FinBERT-chinese 或 YangYi financial sentiment）：
- 情感极性 Score（-1 ~ +1）
- 情感波动率（过去 4 小时标准差）
- 关键词命中数（涨停、利好、减持、监管、爆雷）
- **情感加速度** = 当前 1h 均值 - 前 1h 均值（最重要）

#### 量价侧：日内微观结构特征
从 1 分钟 K 线中提取 20 维特征：
- **动量类**：过去 30/60/120 分钟收益率、开盘跳空幅度
- **波动类**：日内已实现波动率（Realized Vol）、振幅衰减率
- **量能类**：成交量较前 5 日均量比率、大单净流入估算、委比变化
- **结构类**：外盘/内盘比率、价格-成交量相关系数、MACD 分钟级金叉死叉
- **情绪类**：涨跌停家数（同概念）、北向资金 15 分钟净流入

#### 融合侧：时间对齐
以 **30 分钟时间桶** 为单位对齐：
- 每个桶 = `[文本向量(256d) + 量价特征(20d) + 宏观/板块(4d)]`
- 输入序列长度：过去 **16 个时间桶**（约 1.5 个交易日）

### 2.4 模型架构：轻量 Dual-Stream Transformer

```
                      ┌─────────────────┐
文本序列 (16×256) ───→│  Text Encoder   │──→ Text Embedding (16×64)
                      │  (2层 Transformer)│
                      └─────────────────┘
                                           ┐
                      ┌─────────────────┐  │  Cross-Attention
量价序列 (16×24)  ───→│ Price Encoder   │──→ Price Embedding (16×64)
                      │  (2层 Transformer)│  │
                      └─────────────────┘  ┘
                                           ↓
                              Concat + 2层 MLP
                                           ↓
                    输出：未来1日异动概率 [0~1]
                    输出：方向判断 {大涨/震荡/大跌}
```

**Cross-Attention 融合**：当突发新闻出现时，文本流的高激活向量会 query 到量价流中对应时间点的成交量爆量特征，识别「舆情与资金的共振」。

**训练策略**：
- 标签：未来 1 日收益率分三类（大涨 >+3%，大跌 <-3%，震荡）
- 损失函数：Focal Loss（解决类别不平衡）
- 时序分割：严格按月滚动训练

### 2.5 展示输出（Markdown 样例）

```markdown
### 🤖 AI 异动预警（舆情+量价融合模型）
- **明日异动概率**：72%（高）
- **情绪-资金共振度**：⚡⚡⚡⚡⚪ 4/5
- **关键触发因子**：
  1. 雪球关注度 2 小时内上升 340%
  2. 概念板块尾盘 30 分钟大单净流入占比 18%
  3. 股吧情感极性从 -0.1 跳升至 +0.67
- **模型建议**：明日早盘若高开 >2% 且缩量，建议减仓止盈。
```

## 三、引擎 B：财务趋势预警（LSTM）

### 3.1 业务目标
预测 **下一季度财报披露时，核心财务指标的变化方向**：
- ROE 变化方向（↑ / → / ↓）
- 营收增长率变化方向
- M-Score 操纵风险是否上升
- 综合财务健康度得分（0-100）

### 3.2 数据输入
直接复用现有 **18 步财务分析结果**：
每个季度 = 一个特征向量（约 18-30 维）：
```
[ROE, GrossMargin, RevenueGrowth, DebtRatio, CashRatio,
 MScore, CoreProfitMargin, AssetTurnover, InventoryTurnover,
 FixedAssetRatio, InterestCoverage, ...]
```
**关键处理**：使用 **环比变化率** 和 **行业 percentile 排名**，并加入季度虚拟变量（Q1/Q2/Q3/Q4）。

### 3.3 模型架构：BiLSTM + Attention

```
过去 8 个季度财务向量 ──→ BiLSTM (64 hidden) ──→ 
                                      │
                                      ↓
                              Self-Attention ──→ 
                                      │
                                      ↓
                           ┌──────────┴──────────┐
                           ↓                     ↓
                      任务头 A（ROE）        任务头 B（营收）
                      任务头 C（M-Score）     任务头 D（健康度）
```

**为什么用 BiLSTM**：财务季度数据序列极短（8 个季度），Transformer 容易过拟合，BiLSTM 参数量小更稳定。

### 3.4 标签与训练
- 正样本：下一季度 ROE 环比提升 >1pp，且营收增长环比提升
- 负样本：下一季度 ROE 环比下降 >1pp，或 M-Score 操纵风险上升
- 中性：其余
- 数据增强：高斯噪声、随机 Mask 模拟数据缺失

### 3.5 展示输出（Markdown 样例）

```markdown
### 📉 财务趋势预警（LSTM 季度预测）
| 指标 | 预测方向 | 置信度 | 关注点 |
|------|---------|--------|--------|
| ROE | ↓ 下滑 | 78% | 毛利率承压可能传导至净利率 |
| 营收增长 | ↑ 回升 | 65% | 新产能释放贡献增量收入 |
| M-Score | → 持平 | 82% | 暂无财报操纵风险加剧信号 |
| **综合健康度** | **72/100** | — | 较上季度预计下降 6 分 |
```

## 四、两引擎协同决策

| 维度 | 引擎 A（高频） | 引擎 B（低频） |
|------|---------------|---------------|
| 时间尺度 | 1~3 个交易日 | 1 个季度 |
| 驱动因素 | 情绪、资金、事件 | 基本面、财务质量 |
| 建议用途 | 择时、仓位管理 | 选股、长期持有信心 |
| 置信度标识 | 低/中/高 + 5 柱信号 | 方向概率 + 归因解释 |

**冲突处理示例**：
- A 高、B 低：短期情绪推动，中期基本面存疑，建议做 T 或减仓。
- A 低、B 高：情绪冰点，基本面向上，适合逢低布局。

## 五、桌面端实现方案

### 5.1 技术栈
- **模型训练**：离线 Python（PyTorch / PyTorch Lightning）训练，导出 ONNX
- **桌面端推理**：Go 端调用 `onnxruntime-go`，完全本地运行，无网络依赖
- **文本模型**：轻量 TextCNN（ONNX 约 5MB），避免引入 500MB 的 FinBERT
- **数据缓存**：30 分钟级量价数据存本地 SQLite

### 5.2 模块集成位置
- **引擎 A**：嵌入报告模块 13（社交媒体情绪监控）与新增模块 9.5（AI 异动预警）
- **引擎 B**：嵌入报告模块 3（基本面分析）末尾或模块 1（执行摘要）

### 5.3 文件规划
```
ml_models/
├── engine_a_sentiment/
│   ├── train.py              # TextCNN + Dual-Stream Transformer 训练
│   ├── export_onnx.py        # ONNX 导出
│   └── sentiment_onnx/       # 情感子模型 ONNX
├── engine_b_financial/
│   ├── train.py              # BiLSTM + Attention 训练
│   ├── export_onnx.py        # ONNX 导出
│   └── financial_lstm.onnx   # 财务预警模型
└── README.md
```

## 六、开发顺序

1. **引擎 B 优先**：数据已具备（18 步财务指标），最快验证。
