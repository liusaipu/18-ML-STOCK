# Stock Analyzer 手机版产品规划

> **核心定位**：「每日 3 分钟，持仓无忧」
> 
> 用「健康体检」的隐喻降低认知门槛，用「智能卡片」替代长篇报告，用「actionable 建议」指导买卖决策。

---

## 一、产品定位：Stock Health

### 核心隐喻：股票 = 体检对象
- 每只自选股每天做一次"体检"
- 输出一个 **0-100 健康分**（由 A-Score 反向计算：100 - A-Score）
- 用 🟢🟡🔴 三色状态直观呈现
- 关键变化用「体检异常项」的方式表达

### iPhone 设计原则
| 原则 | 实现方式 |
|------|---------|
| **单手持握** | 所有核心操作在屏幕下半部完成 |
| **一眼即懂** | 首页只放最重要的 3 个数字：健康分、今日涨跌、关键警报数 |
| **零学习成本** | 交互借鉴 Apple Health、天气 App |
| **即时反馈** | 手势操作配合 Taptic Engine 震动反馈 |
| **系统级入口** | 支持 iOS 小组件、灵动岛、锁屏实时活动 |

## 三、页面 1：首页（Today）—— 每日自选股体检

### 3.1 视觉设计

**顶部区域：日期与总览**
```
2026年4月14日 星期二          [设置⚙️]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
您的 8 只自选股中
🟢 5 只健康  🟡 2 只关注  🔴 1 只警报
```

**核心交互：股票卡片流**
- 垂直滑动 = 浏览不同股票
- 左右滑动 = 切换同一只股票的「昨日/今日」对比
- 向下滑动 = 刷新当前股票数据（配合下拉动画 + 震动）

**单张卡片设计：**
```
┌─────────────────────────────┐
│  贵州茅台    600519.SH  🔴  │
│  健康分: 72  │  今日 -1.2%  │
│                             │
│  ┌─────────────────────┐    │
│  │   [迷你趋势图]      │    │
│  │   近30日健康分走势   │    │
│  └─────────────────────┘    │
│                             │
│  ⚠️ 2 项异常需关注:         │
│  • A-Score 升至 28（低风险→中风险）│
│  • 毛利率连续2季度下滑       │
│                             │
│  [查看完整体检报告 >]         │
└─────────────────────────────┘
```

### 3.2 手势操作（Taptic 反馈）

| 手势 | 操作 | 反馈 |
|------|------|------|
| 垂直滑动 | 切换下一只股票 | 轻微震动 |
| 向下滑动 | 刷新当前股票 | 弹簧动画 + 成功震动 |
| 长按卡片 | 进入多选模式（批量操作） | 重度震动 |
| 从底部上滑 | 展开该股票的详细体检报告 | 平滑过渡 |

## 四、页面 2：体检报告（Health）—— 卡片流详情

### 4.1 动态卡片排序

**不是固定 16 模块，而是根据股票状态智能排序：**

```
优先级 1: 关键警报卡（如果有 🔴）
优先级 2: 投资建议卡
优先级 3: 风险评分卡（A-Score）
优先级 4: 基本面摘要卡
优先级 5: 行业对比卡
优先级 6: 技术面卡
优先级 7: 可比公司卡
优先级 8: 政策匹配卡
优先级 9: 舆情监控卡
```

**每张卡片统一结构：**
```
┌─────────────────────────────┐
│  🏥 风险评分              📋│  ← 右上角复制按钮
│  ─────────────────────────  │
│  A-Score: 28 (中等风险)      │
│                             │
│  [仪表盘可视化]              │
│     ◠◡◠ 指针偏黄            │
│                             │
│  💡 结论: 财报可信度下降，   │
│     需关注应收账款增速       │
│                             │
│  [问 AI 🤖]  [分享 →]        │
└─────────────────────────────┘
```

### 4.2 卡片内手势

| 手势 | 效果 |
|------|------|
| 左滑 | 收藏该结论（⭐️ 存入「我的收藏」） |
| 右滑 | 生成分享图（带结论 + 关键数据 + 品牌水印） |
| 点击卡片 | 展开详细内容（从卡片中央放大过渡） |
| 双击 | 快速标记为「已读」（卡片颜色变淡） |

### 4.3 可视化设计（iPhone 友好）

**仪表盘（Gauge）替代文字：**
```
    ROE 15.2%
      ◠◠◠
    ┌─────┐
    │ ◉───┤  绿色区域（优秀）
    └─────┘
    行业均值 12.5%
```

**迷你趋势图（Sparkline）：**
- 近 30 日健康分曲线
- 下方标注关键事件点（如财报发布日）

**雷达图（仅限 Pro）：**
- 6 维度：盈利、成长、风险、估值、技术、政策
- 点击维度可切换为柱状图

## 五、AI 融合：渐进式对话入口

### 5.1 「问 AI」设计

**位置：** 每张卡片底部固定悬浮一个半透明 AI 按钮
**交互：** 点击后从卡片底部滑出快捷问题 + 输入框

**快捷问题（基于上下文动态生成）：**
```
┌─────────────────────────────┐
│  🤖 关于这张卡片             │
│  ─────────────────────────  │
│  "这个评分下周会变吗？"       │
│  "和同行业比差距多大？"       │
│  "历史出现过类似情况吗？"     │
│  ─────────────────────────  │
│  [输入你的问题...]           │
└─────────────────────────────┘
```

**AI 回答形式：**
- 不超过 3 句话
- 配一个关键数据或迷你图
- 可直接收藏或分享

### 5.2 场景化主动提示

当用户停留某只股票超过 10 秒，或反复查看某张卡片时，AI 主动弹出：
> "检测到你对这只股票的风险评分很关注。需要我对比一下它过去 3 次财报季的表现吗？"

## 六、推送机制：智能、克制、有效

### 6.1 推送类型

| 类型 | 触发条件 | 推送内容 | 时间 |
|------|---------|---------|------|
| **早报** | 每日固定 | "今日 8 只自选中，1 只触发警报" | 用户自定义（默认 8:30） |
| **红色警报** | 健康分单日下降 >15 或 A-Score 突破 60 | 🚨 强震动 + 锁屏横幅 + 灵动岛 | 即时（非静默时段） |
| **黄色关注** | 健康分下降 5-15 或 1-2 项指标恶化 | 汇总在早报中，不单独打扰 | 早报时间 |
| **事件提醒** | 财报发布前 1 天、股东大会等 | "贵州茅台将于明日发布年报" | 前一日 18:00 |
| **周报** | 每周五收盘后 | 本周自选股健康分变化汇总 | 周五 19:00 |

### 6.2 静默时段

```
默认: 23:00 - 07:00
期间: 只计算数据，不推送任何通知
红色警报在静默时段产生 → 延迟到 07:30 集中推送
```

### 6.3 iOS 专属特性

- **锁屏小组件**：显示今日警报数量
- **灵动岛 / 实时活动**：刷新数据时显示进度
- **Siri 快捷指令**："看看我的自选股今天怎么样"

## 八、商业模式

### 8.1 Free 版
- 5 只自选股
- 基础健康分 + 3 张核心卡片
- 每日早报
- 7 天历史回溯

### 8.2 Pro 版（订阅制）
- 无限自选股
- 完整卡片流 + 雷达图
- 365 天历史档案
- 自定义警报规则
- AI 深度问答无限制
- 家庭共享（2-3 人）

### 8.3 增长机制
- **新用户**：14 天 Pro 全功能试用
- **邀请奖励**：邀请 1 人 = +1 个月 Free 版股票额度
- **成就系统**：「连续 30 天查看早报」获得 7 天 Pro 体验

## 九、技术架构建议

### 服务端
- **Go 后端** 封装成 RESTful API（复用现有 18 步引擎）
- **分析任务异步化**：复杂计算提交到队列，完成后推送给手机
- **数据缓存策略**：
  - Redis：热点股票实时行情
  - PostgreSQL：用户数据、历史评分、体检记录

### 客户端
- **iOS 原生开发**（SwiftUI + Combine）
  - 动画流畅，手势响应精准
  - 完美适配灵动岛、小组件
- **Flutter** 备选（如果同时要 Android）
  - 但 iPhone 体验会略逊于原生

### 数据同步
- 每天早上 6:00 服务端批量计算所有用户自选股的体检报告
- 用户打开 App 时，优先展示缓存，后台静默刷新
- 弱网环境下，保证首页卡片 2 秒内可见

## 十一、关键设计原则总结

> **"不是把桌面版变小，而是把体检体验做到极致。"**

1. **减法思维**：默认只展示最重要的 3 张卡片，其他折叠
2. **行动导向**：每张卡片必须有 1 个明确结论 + 1 个行动建议
3. **情感化设计**：健康分上升时用庆祝动画，警报时用紧迫感配色
4. **iOS 原生体验**：从手势到震动到小组件，融入 Apple 生态
