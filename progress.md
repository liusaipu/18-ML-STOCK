- [x] 验证修复效果

**问题分析**:
用户报告更新行业数据时出错：`exit status 1`，错误信息乱码

**根本原因**:
1. Python 脚本依赖 `akshare.stock_industry_category_cninfo()` 外部API，该接口不稳定
2. Windows 控制台编码问题导致中文乱码
3. 财务数据格式理解错误（实际是 `{科目: {日期: 值}}` 而非 `{日期: {科目: 值}}`）

**修复方案**:
- 改为从本地已下载的股票财务数据计算行业均值，不再依赖外部API
- 修复 Windows 控制台编码问题（设置 stdout/stderr 为 utf-8）
- 适配正确的财务数据格式
- 添加字段别名支持（如 "归属于母公司股东权益合计"/"所有者权益合计"/"股东权益合计"）
