# 18步法自选股票分析系统

这个项目需要相对持续的开发，项目目录位于：

```
~/.openclaw/workspace/stock-analyzer
```

主要实现18步法自选股票分析的系统，使用 Wails（Go + Web 前端）方案来实现，能跨 macOS、Windows 和 Linux 运行。

## 主要功能点

- 主要想实现对自选股票的长期跟踪，这个自选列表可以增删改查。
- 包括 A 股市场沪、深、创业板、科创板、北交所，还有港股。
- 能对自选列表里的单只股票进行"十八步财报分析法"进行分析处理，生成分析报告，报告采用 `.md` 格式输出。
- "18步财报分析法"具体方法及步骤参考如下 URL：
  ```
  https://ima.qq.com/note/share/_AwbPbqO9gCYLZwWa19uMg?channel=4
  ```
- 里面涉及表格汇总及计算需要从同花顺下载三张表 CSV 格式。
- 自选列表
  - 最多自选 100 只，显示代码和市场，类似 `600001.SH`。
  - 添加框可以根据输入实时提示可选代码，下拉 10 只左右。
  - 单击自选列表某只，右侧显示基本信息与操作区。
  - 可以针对单只股票做 18 步分析法进行分析，右侧显示生成的 markdown 格式内容；支持多次生成结果按日期排列与切换查看。

## 一、对 "十八步财报分析法" 的核心理解（基于 18analysis.txt）

18 步分析可归纳为 4 个层次：

1. **基础层（第1步）**：审计意见与审计师资质。非标准无保留意见或事务所非十大所（普华永道、安永、毕马威、德勤、天健、立信、大华、容诚、天职国际、信永中和），直接排除。
2. **结构层（第2-8步）**：资产规模、负债结构（资产负债率、准货币资金-有息负债）、行业地位（应付预收-应收预付，即"两头吃"能力）、资产质量（固定资产/投资类资产/应收款/存货/商誉/MScore）。判断公司财务健康度和爆雷风险。
3. **盈利与营运层（第9-14步）**：成长性（营收增长率）、毛利率、存货周转率/总资产周转率、费用管控（期间费用率占毛利率比）、主营利润质量（主营利润率、主营利润占营业利润比）。判断是不是"好生意"。
4. **现金流与回报层（第15-18步）**：净利润含金量（经营活动现金流净额/净利润）、ROE、归母净利润增长率、资本支出（购建长期资产现金支出/经营现金流）、分红可持续性（分配股利现金支出/经营现金流）。判断是不是"好公司"且"对股东好"。

**最终输出要求**：
- 分步骤详细计算（指标计算方法、判断标准、计算过程、分析结论）。
- 十八步财务维度达标测评汇总表（分年度、分维度，标注未达标项）。
- 行业横向对比分析（可比公司均值/最大最小值对比）。
- 百分制打分及优化建议（18项核心维度，每项5.56分，按偏离程度扣1-3分）。

## 二、技术架构

- **Wails v2** (Go + React + TypeScript)
- **Go 后端**：CSV 解析、财务计算引擎、本地文件存储、Markdown 报告生成、东方财富网络下载
- **React 前端**：三栏布局（自选列表 | 股票信息/操作/亮点风险 | 报告展示），支持深色/浅色主题切换
- **本地运行**：财报数据存储在本地磁盘，适合敏感数据管理；同时支持从东方财富免费接口网络下载

## 三、数据层设计

### 3.1 本地存储目录结构

```
~/.config/stock-analyzer/

├── stocks.json                 # 内置A股+港股代码库
├── comparables.json            # 可比公司配置（按主股票代码分组）
├── data/
│   └── {symbol}/               # 单只股票当前生效的财报数据
│       ├── balance_sheet.json
│       ├── income_statement.json
│       ├── cash_flow.json
│       ├── profile.json        # 股票基本资料缓存（行业、上市日期、PE/PB等）
│       └── history/            # 历史版本（自动保留最近3批）
│           ├── 2026-04-03T10:00:00Z/
│           │   ├── balance_sheet.json
│           │   ├── income_statement.json
│           │   ├── cash_flow.json
│           │   └── meta.json   # 记录来源、时间、年份
│           └── ...
├── comparables/
│   └── {symbol}/               # 可比公司财报数据缓存
│       ├── balance_sheet.json
│       ├── income_statement.json

│       └── meta.json
└── reports/
    └── {symbol}/
        ├── 2024-01-15_10-30-00_分析报告.md
        ├── 2024-06-20_14-20-00_分析报告.md
        └── ...
```

### 3.2 可比公司配置设计

- **存储格式**：`comparables.json`，按主股票代码映射到可比公司代码列表。
- **交互方式**：中栏增加"可比公司"小面板，支持搜索添加（3~5家）和移除。
- **自动下载**：添加后后台自动调用现有下载逻辑获取可比公司财报，缓存到 `comparables/{symbol}/`。
- **分析触发**：执行 18 步分析时，自动读取可比公司缓存数据，计算关键指标均值并插入报告。

### 3.3 股票代码库

- 内置完整的 A股（沪、深、创业板、科创板、北交所）+ 港股代码表。
- 当前共 **8,239 只**（沪市2,308、深市2,886、北交所303、港股2,742）。
- 统一格式：`代码.市场`，如 `600519.SH`、`000001.SZ`、`920000.BJ`、`00700.HK`。
- 添加自选时支持按代码/名称模糊匹配，下拉显示前10条。

### 3.4 CSV 导入策略

- **数据来源**：用户在同花顺 F10/iFinD 手动下载三张表 CSV，通过软件"导入财报"功能关联到指定股票。
- **CSV 解析器特点**：
  - 宽容解析同花顺不同版本导出的表头差异。
  - 自动识别空值 `--`、去除 `(元)` 后缀、标准化科目名称。
  - 自动根据文件名（debt/benefit/cash）或文件内容判断报表类型。
  - 解析后保存为 JSON 格式到对应股票的数据目录。
- **历史版本保留**：每次导入或网络下载成功后，自动将当前3张表归档到 `history/{timestamp}/`，并记录 `meta.json`（来源类型、来源网站、时间、年份），最多保留最近 **3 批**。

### 3.5 网络下载策略

- **数据源**：东方财富 HSF10 API。
- **自动探测**：通过尝试 companyType 1~8 自动识别正确的报表类型。
- **字段映射**：完整的资产负债表/利润表/现金流量表字段映射，涵盖研发支出、折旧、分红、递延所得税等。
- **多源校验**：下载后自动对接 `datacenter-web` API 校验关键科目（总资产、总负债、所有者权益、货币资金），差异为 0% 即通过。
- **股票基本资料**：同时对接东方财富行情接口 `push2.eastmoney.com/api/qt/stock/get`，获取行业、上市日期、总股本、总市值、市盈率(动)、市净率，缓存 7 天。
- **测试验证**：韦尔股份（603501）2012-2025 年年报数据已完整下载并校验通过。

### 3.6 已准备的测试数据

- **韦尔股份（603501.SH）** 2012-2025 年三张表 CSV，覆盖重大资产重组前后数据，可用于验证分析引擎对异常数据的处理能力。

## 五、后端架构与 API

### 5.1 核心模块

- `app.go`：Wails 主应用，暴露前端绑定方法。
- `storage.go`：本地文件读写（自选列表、财报数据、报告目录、历史版本归档、股票基本资料缓存、可比公司配置与缓存）。
- `csvparser.go`：同花顺 CSV 宽容解析器。
- `analyzer/`：18步分析引擎（已完整实现）。
  - `data.go`：财务数据结构、科目归一化。
  - `steps.go`：18步逐条计算。
  - `mscore.go`：Beneish M-Score 完整模型。
  - `evaluator.go`：扣分规则、百分制评分。
  - `report.go`：14模块专业 Markdown 报告生成。
  - `engine.go`：分析入口编排。
- `downloader/`：东方财富网络下载器。
  - `eastmoney.go`：HSF10 API 并发下载、股票基本资料获取。
  - `mapping.go`：科目映射。
  - `validator.go`：datacenter-web 校验。
  - `storage.go`：下载数据本地保存。

## 六、报告输出格式

分析报告采用 **14 模块专业投资分析框架**，参考迈瑞医疗报告样本结构：

| 模块 | 状态 | 说明 |
|---|---|---|
| 1. 执行摘要 | ✅ | 多维度评分汇总、关键指标速览、综合评级与建议 |
| 2. 换手率深度分析 | ⏳ 占位 | 待接入行情接口 |
| 3. 公司基本面分析 | ✅ | 全年核心财务数据、近5年趋势、18步分类达标、核心风险点 |
| 4. 行业横向对比分析 | 📝 规划中 | **轻量版**：用户自选 3~5 家可比公司，对比最新一年关键指标（ROE、毛利率、营收增长率、资产负债率、净利润现金含量、M-Score）的均值/最高/最低/排名；**完整版**：多年度趋势对比（阶段7扩展） |
| 5. 十五五政策匹配度评估 | ⏳ 占位 | 待扩展产业政策库 |
| 6. Tushare实时数据 | ⏳ 占位 | 待接入行情接口 |
| 7. RIM剩余收益模型估值 | ✅ | 基于现有 ROE 的简化版三情景估值（悲观/基准/乐观） |
| 8. 技术面分析 | ⏳ 占位 | 待扩展技术指标 |
| 9. ML机器学习预测 | ✅ | 基于财务正负向因子的简易推断 |
| 10. 智能选股6大条件 | ✅ | ROE≥15%、毛利率≥40%、营收增长≥10%、M-Score≤-2.22、资产负债率≤60%、净利润现金含量≥100%、准货币资金-有息负债≥0 |
| 11. 芒格逆向思维检查 | ✅ | 逆向三问 + 评分 |
| 12. 巴菲特-芒格投资检查清单 | ✅ | 7项核心检查 |
| 13. 社交媒体情绪监控 | ✅ | 对接东财研报接口做情绪分析，输出情绪得分、热度指数与舆情摘要 |
| 14. 综合投资建议 | ✅ | 加权评分汇总 + 策略建议 |
| 15. 结论与附录 | ✅ | 核心结论、关键数据速查、投资逻辑总结、免责声明 |

## 八、关键风险与应对

1. **CSV 格式不一致**：同花顺不同年份表头可能有差异。应对：解析器支持字段别名映射，并持续用真实 CSV 迭代修正。
2. **MScore 计算复杂**：涉及 8 个细分指标、跨年度数据。应对：已单独写单元测试，用韦尔股份数据手工验算通过。
3. **重大资产重组导致数据突变**：如韦尔股份 2019 年商誉、资产暴增。应对：分析引擎输出时增加异常值检测（M-Score 等模型已覆盖）。
4. **网络数据源稳定性**：东方财富接口偶发限流或空返回。应对：已加入自动 companyType 探测、datacenter-web 校验、股票基本资料缓存7天，异常时提示用户改用 CSV 导入。
5. **可比公司批量下载限流**：横向对比需要同时下载多家公司财报，可能触发限流。应对：采用串行下载+本地缓存策略，避免并发高峰；若某只下载失败，报告中标记为"数据缺失"，不阻塞整体分析流程。
