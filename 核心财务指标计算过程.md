# 核心财务指标计算过程溯源功能设计

## 背景

当前系统生成的分析报告中，部分核心财务指标（如 ROE、毛利率等）的计算口径可能与主流财经网站存在差异。例如：

- **成都华微 2024 年 ROE**：系统计算为 6.32%，这与东方财富等平台的展示值一致，但原因是该公司 2024 年内无大额增发/分红，导致"全面摊薄 ROE"与"加权平均 ROE"恰好接近。
- **根本差异**：系统当前使用的是 **全面摊薄 ROE**（期末净资产），而主流平台通常展示 **加权平均 ROE**。当公司年内存在增发、回购或大额分红时，两种口径差异会非常明显。

为了让用户对系统生成的每一个判断和数值都有"可验证、可追踪"的信心，需要增加一个**计算过程溯源**功能。

---

## 设计目标

1. **透明性**：每个计算指标都能追溯到原始科目、数值、公式和分步计算过程。
2. **非侵入性**：不干扰现有报告 Markdown 的生成和展示逻辑。
3. **按需触发**：用户在需要时才查看计算明细，避免报告正文过于冗长。
4. **可扩展性**：从核心指标（ROE、毛利率）试点，最终覆盖 18 步分析的全部计算步骤。

---

## 数据结构设计

### Go 后端结构

```go
// CalcTrace 单个指标的计算溯源记录
type CalcTrace struct {
    Indicator string                 `json:"indicator"` // 指标名，如 "ROE"
    Year      string                 `json:"year"`      // 所属年份
    Formula   string                 `json:"formula"`   // 公式文本
    Inputs    map[string]InputValue  `json:"inputs"`    // 输入原始数据
    Steps     []CalcStep             `json:"steps"`     // 分步计算过程
    Result    float64                `json:"result"`    // 最终结果
    Note      string                 `json:"note"`      // 口径说明/备注
}

type InputValue struct {
    Source string  `json:"source"` // 数据来源：资产负债表 / 利润表 / 现金流量表
    Item   string  `json:"item"`   // 科目名
    Year   string  `json:"year"`   // 年份
    Value  float64 `json:"value"`  // 原始数值（元）
    Note   string  `json:"note"`   // 备注，如"取年末值"
}

type CalcStep struct {
    Desc  string  `json:"desc"`  // 步骤描述
    Expr  string  `json:"expr"`  // 表达式文本
    Value float64 `json:"value"` // 该步结果
}
```

### StepResult 扩展

在现有的 `StepResult` 中增加 `Traces` 字段：

```go
type StepResult struct {
    StepNum   int                    `json:"stepNum"`
    StepName  string                 `json:"stepName"`
    Conclusion string                `json:"conclusion"`
    YearlyData map[string]map[string]any `json:"yearlyData"`
    Pass      map[string]bool        `json:"pass"`
    Traces    []CalcTrace            `json:"traces"` // 新增：计算溯源
}
```

## 后端实现方案

### 1. 计算时顺带生成 Trace

以 `step16ROE` 为例，在计算 ROE 的同时，生成对应的 `CalcTrace`：

```go
trace := CalcTrace{
    Indicator: "ROE",
    Year:      year,
    Formula:   "归母净利润 / 期末归母所有者权益合计 × 100%",
    Note:      "当前使用全面摊薄ROE。主流平台（东方财富/同花顺）展示的是加权平均ROE，若年内存在增发/分红，两者会有差异。",
}

trace.Inputs["profit"] = InputValue{Source:"利润表", Item:"归母净利润", Year:year, Value:profit}
trace.Inputs["equity"] = InputValue{Source:"资产负债表", Item:"归母所有者权益合计", Year:year, Value:equity, Note:"取年末值"}

trace.Steps = append(trace.Steps, CalcStep{
    Desc:  "计算ROE",
    Expr:  fmt.Sprintf("%.0f / %.0f × 100%%", profit, equity),
    Value: roe,
})

trace.Result = roe
result.Traces = append(result.Traces, trace)
```

## 前端交互方案

### 推荐方案：Hover 触发 + 右侧抽屉

在报告展示区的"核心指标一览"和"核心财务指标趋势"表格中，为每个数值单元格增加交互：

1. **鼠标悬停**在数值上时，单元格背景微微高亮，并显示一个小的 "🔍" 或 "?" 图标。
2. **点击**后从右侧滑出一个抽屉（Drawer），展示该指标的完整 `CalcTrace`。
3. **抽屉内容布局**：

```
┌─────────────────────────────────────┐
│ ROE (2024年) 计算过程          [×]  │
├─────────────────────────────────────┤
│ 公式                                │
│ 归母净利润 / 期末归母所有者权益合计 × 100%│
├─────────────────────────────────────┤
│ 原始数据                            │
│ • 归母净利润（利润表，2024）        │
│   ¥120,000,000                      │
│ • 归母所有者权益合计（资产负债表，2024）│
│   ¥1,900,000,000  [取年末值]        │
├─────────────────────────────────────┤
│ 计算步骤                            │
│ 1. 120,000,000 / 1,900,000,000      │
│    = 0.06315789                     │
│ 2. × 100% = 6.32%                  │
├─────────────────────────────────────┤
│ 💡 口径说明                         │
│ 当前使用全面摊薄ROE。如需与东方财富│
│ 对比，注意对方展示的是加权平均ROE。 │
└─────────────────────────────────────┘
```

### 备选方案

- **底部折叠面板**：在报告内容下方增加一个默认收起的"📐 计算过程溯源"区域，一次性列出所有指标。适合打印/导出场景，但信息量大时容易淹没重点。
- **Markdown 超链接**：在生成报告时把关键数字变成链接（如 `[6.32%](#trace-roe-2024)`），点击后前端拦截并弹窗。会轻微污染 Markdown 文本的纯净性。

## 实施步骤

### Phase 1：数据结构 + 试点步骤
1. 在 `analyzer` 包中定义 `CalcTrace`、`InputValue`、`CalcStep` 结构体。
2. 修改 `StepResult`，增加 `Traces []CalcTrace`。
3. 先为 `step16ROE` 和 `step10GrossMargin` 生成 Trace，验证数据结构是否合理。
4. 前端增加"溯源抽屉"组件（`TraceDrawer`）。
5. 在"核心指标一览"表格中为 ROE 和毛利率的数值绑定点击事件，弹出抽屉。

### Phase 2：推广到全部 18 步
1. 依次为剩余的 16 个步骤生成 Trace。
2. 将所有涉及计算的指标数值都加上可点击的溯源入口。
3. 优化抽屉中的数值格式化（自动选择 元/万元/亿元）。

### Phase 3：增强对比功能（可选）
1. 在 ROE 的 Trace 中，尝试同时计算并展示"加权平均 ROE"的近似值（需要读取期初净资产）。
2. 如果系统未来接入了更多季度的明细数据，可以实现更精确的加权平均 ROE。

## 当前发现的问题与行动计划

| 问题 | 原因 | 行动 |
|---|---|---|
| ROE 与部分主流网站不一致 | 口径差异：全面摊薄 vs 加权平均 | 在 Trace 中明确标注口径；未来考虑增加加权平均 ROE 计算 |
| 用户无法验证系统计算结果 | 缺少原始数据展示和公式说明 | 立即启动 Phase 1，优先完成 ROE / 毛利率溯源 |
